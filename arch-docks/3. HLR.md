# 1. Стейкхолдеры

|Роль                   |Цель                                                           |
|-----------------------|---------------------------------------------------------------|
|Пользователь (User)    |Получение информации о погоде и качестве воздуха               |
|Администратор (Admin)  |Добавление, изменение и удаление информации о доступныхгородах |


# 2. Бизнес-цели

1. Разрабротать микросервисный бэкенд, чтобы попрактиковаться в проектировании архитектуры
2. Разработать веб приложение с использование микрофронтенд-архитектуры, чтобы набить руку
3. Самостоятельно реализовать API-Gateway
4. Собрать архитектурные артефакты
5. Попрактиковаться в настройке CI/CD
6. Попрактиковаться в использовании k8s

# 3. High-Level Requirements

## Функциональные

[HLR-01] Пользователь может просматривать погоду через адаптивный web-интерфейс
[HLR-02] Администратор может аправлять списком доступных городов через специальную панель
[HLR-03] Для просмотра погоды используется внешний API

## Нефункциональные

[HLR-04] Микросервисы общаются друг с другом при помощи JSON:API
[HLR-05] API отвечает < 1.5 сек
[HLR-06] Обращение к внешнему API должно быть минимизировано посредством кэширования погодных данных
[HLR-07] Фронтенд-приложение должно быть реализовано как PWA
